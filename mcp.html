MCP: Les API du Futur pour l'IA

Standardiser l'interaction entre IA et Outils

Les 3U : Useful, Usable, Used

Bienvenue à cette présentation sur le Model Context Protocol, ou MCP. Aujourd'hui, nous allons explorer comment ce protocole révolutionne l'interaction entre l'IA et les outils numériques. Le MCP est bien plus qu'un simple standard technique - il représente une nouvelle façon de concevoir la relation entre les agents IA et leur environnement. Au cœur de cette présentation, nous allons découvrir les "3U" du MCP : comment créer des outils qui sont vraiment Utiles (Useful), facilement Utilisables (Usable) et effectivement Utilisés (Used) par les systèmes d'IA.

Contexte et fondamentaux des systèmes agents

Bases des Systèmes Agents

  • Un agent interagit avec un environnement
  • L'environnement fournit des informations observables
  • L'autonomie est la capacité d'agir indépendamment
  • Un agent doit pouvoir modifier son environnement pour être utile
Détails sur les agents et l'environnement: - Un agent est une entité capable d'interagir avec un environnement - L'environnement fournit des informations qui le rendent observable par l'agent - L'environnement offre des capacités permettant à l'agent d'interagir avec lui et de le modifier - Un agent qui ne peut pas modifier son environnement est inutile Détails sur l'autonomie: - L'autonomie est une propriété variable d'un agent - C'est la capacité d'agir de manière indépendante - Il existe plusieurs niveaux d'autonomie - Plus un agent est autonome, moins il a besoin d'interaction humaine - Ne pas confondre agents et workflows : un workflow est un processus d'exécution préprogrammé, un graphe d'exécution

Limites et Défis des Environnements IA

  • Actuellement : fenêtre de chat avec contexte limité
  • Les humains interagissent avec plusieurs environnements numériques
  • Besoin d'élargir la capacité d'action des agents
  • Comment fournir des outils numériques aux agents ?
Limitations actuelles des environnements IA : - L'environnement le plus courant aujourd'hui : la fenêtre de chat - Les LLMs génèrent du texte dans une fenêtre de contexte limitée - L'environnement est fermé, limitant les capacités d'action de l'agent - Pour être vraiment utiles, les agents IA doivent pouvoir interagir avec un environnement plus large Problème des environnements élargis : - Les humains interagissent avec de multiples environnements numériques - Les agents IA sont limités à leur fenêtre de chat - Nous devons trouver comment permettre aux agents d'interagir avec n'importe quel environnement - Il faut leur fournir des outils numériques appropriés

Le Défi de la Liaison Agent-Outil

  • Langages différents : naturel vs API structurées
  • Besoin d'un mécanisme de liaison flexible
  • Éviter le couplage fort avec les SDK actuels
  • Séparation pour créer un écosystème d'outils
  • Nécessité d'un standard !
Problème fondamental de liaison : - Les agents IA comprennent le langage naturel mais pas les API structurées - Nous avons besoin d'un mécanisme de liaison entre la requête en langage naturel et l'appel structuré à l'outil - Les SDK existants créent un couplage fort entre l'agent et l'outil - Idéalement, les outils devraient être séparés de l'agent pour plus de flexibilité Avantages de la séparation agent-outil : - Évolution indépendante : L'outil peut évoluer sans modifier l'agent - Proximité avec le métier : Le propriétaire de l'outil est proche de l'environnement métier - Contrôle des ressources exposées : Le propriétaire décide quelles ressources et actions exposer - Écosystème d'outils : Création d'un Web d'outils utilisables par n'importe quel agent - Pour réaliser cette vision, nous avons besoin d'un standard...

Introduction au MCP

Le Model Context Protocol (MCP)

  • Protocole apparu fin 2023, initié par Anthropic
  • Standard d'exposition des outils pour les IA
  • Le "chaînon manquant" pour transformer assistants en agents

{
  "function_name": "get_weather",
  "description": "Récupère la météo pour un lieu donné.",
  "arguments": [
    {
      "name": "location",
      "type": "string",
      "description": "La ville pour laquelle récupérer les données météo."
    }
  ]
}
Détails sur le MCP : - Contexte : Protocole apparu fin 2023 (Décembre), poussé par Anthropic - Objectif : Standardiser comment les outils logiciels exposent leurs capacités aux intelligences artificielles - Définition : Un standard pour qu'un outil expose des services utilisables par une IA (et non directement par un humain) - Concept Clé : Le "chaînon manquant" pour transformer les assistants (qui assistent) en agents (qui agissent) Sur l'exemple : - Cette description est sérialisée et rendue accessible via un réseau - Elle permet à n'importe quel agent de comprendre et utiliser cet outil - Format basé sur JSON, facile à générer et à parser Concepts clés : - MCP Servers : Les outils exposés selon le standard MCP - Hôte (Host) : L'application qui exécute le modèle de langage - MCP Client : L'implémentation dans l'hôte qui permet de communiquer avec les serveurs MCP - MCP est un standard d'exposition et de communication qui enrichit les agents avec des outils utilisables immédiatement

Le MCP - cadre technique et fonctionnel

Capacités et Affordance pour l'IA

Trois capacités exposées :

  • Ressources : données de l'environnement
  • Actions : fonctions pour agir
  • Prompts : guides de raisonnement

Affordance pour l'IA :

  • Descriptions textuelles
  • Noms de fonctions appropriés
  • Documentation claire
  • Prompts guidant l'utilisation
Capacités exposées par un serveur MCP : - Ressources : Exposer des données issues d'un environnement - Ex: liste de produits, prix d'une action, profil utilisateur - Outils (Actions) : Fournir des fonctions pour exécuter des tâches - Ex: calculer une distance, envoyer un email, réserver un vol - Prompts : Fournir des modèles de texte pour aider l'hôte à raisonner - Ex: templates de description, règles de formatage, bases de connaissances La notion d'affordance pour l'IA : - Affordance : Qualité d'un objet qui suggère sa propre utilisation - Pour un LLM interagissant via texte, l'affordance est fournie par : - La description textuelle de l'outil - Les noms et paramètres des fonctions - La documentation associée - Les prompts exposés - Sans ces indices textuels clairs, le LLM ne peut pas facilement percevoir l'utilité de l'outil - Un outil mal documenté ne sera pas utilisé, même s'il est potentiellement utile

Les 3 U et Aspects Techniques

Les 3 U du MCP :

  • Useful (Utile)
  • Usable (Utilisable)
  • Used (Utilisé)

Aspects Techniques :

  • Basé sur JSON-RPC
  • Extensions : Server-Sent Events
  • Mécanismes de découverte
  • Les API du Futur
Les 3 U du MCP - Useful, Usable, Used : - Useful (Utile) : L'outil résout un problème réel pour l'IA - Usable (Utilisable) : L'outil est accessible via le protocole MCP standard - Used (Utilisé) : L'outil est effectivement choisi et employé par l'IA Les prompts MCP sont essentiels pour garantir ces trois aspects : - Ils guident l'IA sur quand et comment utiliser l'outil efficacement - Ils facilitent l'injection du contexte pertinent - Ils rendent les capacités de l'outil découvrables par l'IA Aspects techniques de MCP : - Format : Communication basée sur JSON-RPC - Extensions futures : Server-Sent Events pour les notifications asynchrones - Découverte : Mécanismes pour que les agents trouvent et comprennent les outils disponibles - Métaphore Centrale : MCP est aux IA ce que les API REST ont été à la transformation digitale → Les API du Futur

MCP vs API REST et Composants fondamentaux

MCP : Les "API REST" pour les IA

Analogie :

  • API REST → humains
  • MCP → agents IA
  • Même rôle catalyseur
  • Affordance adaptée aux IA

Correspondances :

  • Action ≈ POST
  • Resource ≈ GET
  • Prompt = nouveauté
Analogie MCP vs API REST : - Rappel API REST : Catalyseur de la transformation digitale, exposition des capacités logicielles - Parallèle MCP : Même rôle de catalyseur, mais l'affordance est conçue pour les IA - Vision : Changement de paradigme - interaction via des agents IA utilisant les outils MCP, plutôt qu'interaction humaine directe avec l'UI/API Composants fondamentaux du MCP : - Action : Équivalent POST REST. Permet à l'agent de modifier l'état du monde extérieur ou d'effectuer une tâche - Resource : Équivalent GET REST. Permet à l'agent de récupérer de l'information (ex: recherche web, état d'un système) - Prompt : (Souvent sous-estimé mais crucial !) L'outil MCP peut enrichir le prompt de l'IA pour l'aider à décider si, quand et comment utiliser l'outil. C'est la clé de l'affordance pour l'IA

L'Importance Cruciale des Prompts

  • Définissent des templates et workflows pour l'IA
  • Simplifient la collecte de contexte pertinent
  • Rendent les capacités découvrables
  • Transforment potentiel en actions concrètes
  • Guident l'IA sur pourquoi et comment utiliser l'outil
L'importance des prompts dans MCP : - Les prompts MCP permettent aux serveurs de définir des templates réutilisables et des workflows pour l'IA - Avantages clés : * Simplifient la collecte de contexte pertinent (fichiers de code, logs, etc.) * Rendent les capacités du serveur découvrables par l'IA * Transforment un potentiel abstrait en actions concrètes immédiatement disponibles - Le prompt aide l'IA à comprendre : * Pourquoi utiliser cet outil pour résoudre le problème cible * Comment l'utiliser efficacement Exemple d'utilisation d'ontologies avec MCP : - Un graphe de connaissances stocke des relations: "MCP" "améliore" "Agents IA" - Problème : Inconsistance des termes (L'IA pourrait utiliser "améliore" ou "renforce" ou "augmente") - Solution : Prompt MCP avec ontologie * Définit les classes (Protocole, Théorie, Personne) * Standardise les prédicats autorisés (améliore, écrit_par) * Guide l'IA pour maintenir la cohérence

Takeaway 1 : MCP = Les Nouvelles API

Considérez MCP comme l'équivalent des API REST, mais destiné aux IA. C'est le standard émergent essentiel pour l'avenir des applications basées sur l'IA.

Il faut l'intégrer dans nos réflexions stratégiques et architecturales dès maintenant.

Premier point clé à retenir: Le MCP n'est pas juste un autre protocole technique parmi d'autres. Il représente pour les applications IA ce que les API REST ont été pour le web et la transformation digitale. C'est un standard émergent que toutes les grandes entreprises tech adoptent rapidement (Anthropic, OpenAI, Google...). Il sera probablement incontournable dans les années à venir, à mesure que les agents IA se généraliseront. Il est donc crucial de commencer à l'intégrer dans nos réflexions stratégiques et architecturales dès maintenant, pour ne pas prendre de retard sur cette évolution majeure.

Takeaway 2 : Pensez "Affordance pour l'IA"

La valeur d'un outil MCP ne réside pas seulement dans sa fonction, mais aussi dans sa capacité à guider l'IA sur son utilisation via les prompts.

La conception doit viser à rendre l'outil facilement compréhensible par l'IA.

Deuxième point essentiel: Le concept d'affordance - c'est-à-dire la capacité d'un produit à suggérer sa propre utilisation - prend une dimension nouvelle avec les outils pour IA. Pour un humain, l'affordance est souvent visuelle (un bouton qui semble "cliquable", une poignée qui invite à être tirée). Pour une IA, l'affordance est textuelle et conceptuelle. Elle doit comprendre: - Ce que fait l'outil (sa fonction) - Quand l'utiliser (son contexte d'application) - Comment l'utiliser correctement (ses paramètres et contraintes) C'est là que le composant "Prompt" du MCP devient crucial - il ne s'agit pas juste d'une documentation, mais d'un véritable guide d'utilisation pour l'IA. La conception d'outils MCP doit donc intégrer cette dimension dès le départ, en pensant à comment l'outil se présentera à l'IA et comment il la guidera dans son utilisation.

Takeaway 3 : Respectez les 3 U

Un outil MCP réussi doit être Utile (résoudre un problème réel), Utilisable (accessible via le protocole standard), et Utilisé (effectivement choisi par l'IA).

L'affordance pour l'IA est la clé - un outil mal documenté ou sans prompts clairs sera ignoré.

Troisième point fondamental: Les 3 U constituent un cadre simple mais puissant pour évaluer vos outils MCP: - Useful (Utile): L'outil doit résoudre un problème réel et apporter une valeur ajoutée. Un outil qui ne répond à aucun besoin concret ne sera jamais utilisé. - Usable (Utilisable): L'outil doit être techniquement accessible via le protocole MCP standard, avec des interfaces bien définies et des comportements prévisibles. - Used (Utilisé): L'aspect le plus souvent négligé - l'outil doit être conçu de façon à ce que l'IA le choisisse effectivement parmi les options disponibles. Ce dernier point est directement lié à l'affordance: un outil mal documenté, avec des prompts peu clairs ou incomplets, risque d'être tout simplement ignoré par l'IA, même s'il est potentiellement utile et techniquement utilisable. Commencez avec des cas d'usage simples dans le domaine IT, puis étendez progressivement à d'autres cas métier une fois l'expertise acquise.

Contexte et fondamentaux des systèmes agents

Définition d'un Agent et son Environnement

  • Un agent est une entité capable d'interagir avec un environnement.
  • L'environnement fournit des informations qui le rendent observable par l'agent.
  • L'environnement offre des capacités permettant à l'agent d'interagir avec lui et de le modifier.
  • Un agent qui ne peut pas modifier son environnement est inutile.

Le Concept d'Autonomie

  • L'autonomie est une propriété variable d'un agent.
  • C'est la capacité d'agir de manière indépendante.
  • Il existe plusieurs niveaux d'autonomie.
  • Plus un agent est autonome, moins il a besoin d'interaction humaine.
  • Ne pas confondre agents et workflows : un workflow est un processus d'exécution préprogrammé, un graphe d'exécution.

Environnements d'IA Actuels : Limitations

  • L'environnement le plus courant aujourd'hui : la fenêtre de chat.
  • Les LLMs génèrent du texte dans une fenêtre de contexte limitée.
  • L'environnement est fermé, limitant les capacités d'action de l'agent.
  • Pour être vraiment utiles, les agents IA doivent pouvoir interagir avec un environnement plus large que la seule fenêtre de chat.

Le Problème des Environnements Élargis

  • Les humains interagissent avec de multiples environnements numériques.
  • Les agents IA sont limités à leur fenêtre de chat.
  • Comment permettre aux agents d'interagir avec n'importe quel environnement ?
  • Comment leur fournir des outils numériques ?

Problème : Liaison Agent-Outil

  • Les agents IA comprennent le langage naturel mais pas les API structurées.
  • Nous avons besoin d'un mécanisme de liaison entre :
    • La requête en langage naturel de l'agent
    • L'appel structuré à l'outil
  • Les SDK existants créent un couplage fort entre l'agent et l'outil.
  • Idéalement, les outils devraient être séparés de l'agent pour plus de flexibilité.

Avantages de la Séparation Agent-Outil

  • Évolution indépendante : L'outil peut évoluer sans modifier l'agent.
  • Proximité avec le métier : Le propriétaire de l'outil est proche de l'environnement métier.
  • Contrôle des ressources exposées : Le propriétaire décide quelles ressources et actions exposer.
  • Écosystème d'outils : Création d'un Web d'outils utilisables par n'importe quel agent.
  • Pour réaliser cette vision, nous avons besoin d'un standard...

Introduction au MCP

Le Model Context Protocol (MCP)

  • Contexte : Protocole apparu fin 2023 (Décembre), poussé par Anthropic.
  • Objectif : Standardiser comment les outils logiciels exposent leurs capacités aux intelligences artificielles (IA).
  • Définition : Un standard pour qu'un outil expose des services utilisables par une IA (et non directement par un humain).
  • Concept Clé : Le "chaînon manquant" pour transformer les assistants (qui assistent) en agents (qui agissent).

Exemple Concret de Description d'Outil MCP


{
  "function_name": "get_weather",
  "description": "Récupère la météo actuelle pour un emplacement donné.",
  "arguments": [
    {
      "name": "location",
      "type": "string",
      "description": "La ville ou la région pour laquelle récupérer les données météo."
    }
  ]
}

Cette description est sérialisée et rendue accessible via un réseau, permettant à n'importe quel agent de comprendre et utiliser cet outil.

Concepts Clés de MCP

  • MCP Servers : Les outils exposés selon le standard MCP.
  • Hôte (Host) : L'application qui exécute le modèle de langage.
  • MCP Client : L'implémentation dans l'hôte qui permet de communiquer avec les serveurs MCP.
  • MCP est un standard d'exposition et de communication qui enrichit les agents avec des outils utilisables immédiatement.

Le MCP - cadre technique et fonctionnel

Les Capacités Exposées par un Serveur MCP

  • Ressources : Exposer des données issues d'un environnement
    • Ex: liste de produits, prix d'une action, profil utilisateur
  • Outils (Actions) : Fournir des fonctions pour exécuter des tâches
    • Ex: calculer une distance, envoyer un email, réserver un vol
  • Prompts : Fournir des modèles de texte pour aider l'hôte à raisonner
    • Ex: templates de description, règles de formatage, bases de connaissances

Aspects Techniques de MCP

  • Format : Communication basée sur JSON-RPC.
  • Extensions futures : Server-Sent Events pour les notifications asynchrones.
  • Découverte : Mécanismes pour que les agents trouvent et comprennent les outils disponibles.
  • Métaphore Centrale : MCP est aux IA ce que les API REST ont été à la transformation digitale → Les API du Futur.

MCP vs API REST

Analogie : MCP vs API REST

  • Rappel API REST : Catalyseur de la transformation digitale, exposition des capacités logicielles.
  • Parallèle MCP : Même rôle de catalyseur, mais l'affordance (la capacité d'un produit à suggérer sa propre utilisation) est conçue pour les IA.
  • Vision : Changement de paradigme : interaction via des agents IA qui utilisent les outils exposés en MCP, plutôt qu'interaction humaine directe avec l'UI/API.

Composants fondamentaux du MCP

Composants Fondamentaux du MCP

Trois capacités de base exposées par un outil MCP :

  • Action : Équivalent POST REST. Permet à l'agent de modifier l'état du monde extérieur ou d'effectuer une tâche.
  • Resource : Équivalent GET REST. Permet à l'agent de récupérer de l'information (ex: recherche web, état d'un système).
  • Prompt : (Souvent sous-estimé mais crucial !) L'outil MCP peut enrichir le prompt de l'IA pour l'aider à décider si, quand et comment utiliser l'outil. C'est la clé de l'affordance pour l'IA.

Importance et adoption

Importance et Adoption du MCP

  • Standard de Fait : Accepté largement (Anthropic, OpenAI, Google...), devenant le standard d'exposition d'outils pour IA.
  • Écosystème : Permet de construire un écosystème d'outils interopérables, pilotés par l'IA (similaire à l'écosystème API).
  • Catalyseur : Accélère la révolution numérique de l'IA et des agents.

Cas d'usage concrets et actuels

Cas d'Usage Concrets et Actuels

  • Premiers Utilisateurs : Développeurs et professionnels de l'IT (environnement virtuel facile).
    • Exemples d'outils utilisant MCP : Cursor, Cloud Code, Manus...
  • Intérêt de Créer ses Outils MCP :
    • Interagir avec systèmes internes : Base de connaissances (potentiellement au-delà du RAG classique), CI/CD, BDD (ex: SQL en langage naturel).
    • Exposer des capacités métiers spécifiques à l'IA.
  • Écosystème Externe : Grands éditeurs (Salesforce, ServiceNow...) proposent des serveurs MCP pour intégration IA.

Comment aborder le MCP ?

Comment Aborder le MCP ?

  • Fondations Solides : S'appuie sur des savoir-faire éprouvés en ingénierie logicielle (concevoir des outils utiles, utilisables, utilisés).
  • Stratégie de Démarrage :
    • Analyse : Workflows pertinents, cadrage besoin ("Situational Awareness").
    • Conception : Architecture classique (ex: Hexagonale), MCP = un adaptateur/port. Coexistence avec REST naturelle.
    • Développement Itératif : Commencer simple -> Tester -> Mesurer l'usage -> Capitaliser.
  • Nécessité : Sensibilisation des équipes aux enjeux et possibilités.

Évolution future et défis

Évolution Future et Défis

  • Prochaine Étape MCP : "Crafter" des services de qualité (au-delà fonction -> affordance, utilité, utilisabilité pour l'IA).
  • Vers de Vrais Agents : Implémentation des notifications asynchrones MCP (prévu) pour les tâches longues.
  • Agent-to-Agent (A2A) : Protocole futur pour communication inter-agents (jugé plus lointain/prématuré).
  • Défis Actuels :
    • Sécurité & Accréditation : Non spécifiées par défaut dans le protocole (à ajouter au-dessus, comme pour REST/OAuth).
    • Gouvernance : Risque de prolifération -> besoin futur de plateforme/standards internes.
    • Architecture : Déploiement (MCP centralisé vs local) ?

Cas d'usage spécifique : productivité des développeurs

Cas d'Usage : Améliorer la Productivité des Développeurs

  • Problème : IA devs (Copilot...) ont contextes isolés, mémoire limitée à la session/base de code. Manque de contexte partagé de l'équipe.
  • Solution MCP : Serveur MCP exposant une "mémoire partagée" (ex: via Knowledge Graph) pour l'équipe.
  • Bénéfices :
    • L'IA peut interroger ce service (bonnes pratiques, décisions d'archi, guidelines projet...).
    • Co-construction base de connaissance via interactions IA.
    • Améliore cohérence et efficacité des IA individuelles au sein de l'équipe.

Conclusion

MCP : Le Catalyseur de la Prochaine Révolution Numérique

  1. 1ère révolution : Internet (omnicanal, services accessibles depuis le canapé)
  2. 2ème révolution : Smartphone (services numériques nomades, accessibles partout)
  3. Prochaine révolution : Délégation des tâches cognitives à des assistants IA

MCP joue le même rôle catalyseur pour l'ère des agents IA que les API REST pour le web.

Takeaway 1 : MCP = Les Nouvelles API

Considérez MCP comme l'équivalent des API REST, mais destiné aux IA. C'est le standard émergent essentiel pour l'avenir des applications basées sur l'IA.

Il faut l'intégrer dans nos réflexions stratégiques et architecturales dès maintenant.

Takeaway 2 : Pensez "Affordance pour l'IA"

La valeur d'un outil MCP ne réside pas seulement dans sa fonction (Action/Resource), mais aussi dans sa capacité à guider l'IA sur son utilisation (via le composant Prompt notamment).

La conception doit viser à rendre l'outil facilement compréhensible et utilisable par l'IA.

Takeaway 3 : Commencez par les Développeurs

Les cas d'usage les plus immédiats et à forte valeur ajoutée concernent l'amélioration de la productivité des équipes IT (ex: mémoire partagée, interaction CI/CD, BDD...).

C'est un bon point de départ pour expérimenter, itérer et développer l'expertise MCP en interne.

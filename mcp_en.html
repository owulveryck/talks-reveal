The three MCP superpowers explained

Use the "senses" metaphor to explain MCP capabilities:

1. Resources - The eyes of AI

  • Equivalent of GET requests in REST
  • Allows AI to "see" data from the outside world
  • Concrete examples: product list, weather in a city, user profile
  • Without resources, AI remains limited to its internal knowledge

2. Actions - The hands of AI

  • Equivalent of POST requests in REST
  • Allows AI to act and modify the environment
  • Living examples: send an email, place an order, schedule an appointment
  • This is what truly transforms an assistant into an agent

3. Prompts - The augmented brain

  • Unique element to MCP, no REST equivalent
  • Provides guides, templates and knowledge to the AI
  • Examples: how to structure a request, what options are available
  • Often neglected component but absolutely crucial for a tool to be effectively used

Emphasize the synergy: it's the combination of all three that gives MCP its power.

Concrete example: the knowledge graph

Illustrate with a real case:

  • Problem: Without guidance, AI invents different terms each time
    • Session 1: "MCP" "is used by" "Claude"
    • Session 2: "Claude" "implements" "MCP"
    • Result: Inconsistent graph, impossible to query effectively
  • Solution: MCP Prompt that defines a strict ontology
    • Allowed relations: only "implements", "extends", "uses"
    • Entity classes: "Model", "Protocol", "Company"
    • The AI naturally follows these constraints in its interactions

Crucial point to emphasize: Without well-designed prompts, the tool may be perfectly functional but never used - AI won't know when and how to call upon it.

  • Each interaction enriches the common base
  • Best practices are automatically shared

The memory problem of AIs - concretely illustrated

Present this problem with a powerful analogy:

  • Imagine a team where everyone loses their memory every night:
    • Every morning, everyone has to relearn the decisions made yesterday
    • Impossible to capitalize on past experiences
    • Each person rediscovers the same problems in isolation
  • This is exactly what happens with current AI assistants:
    • ChatGPT, Copilot and other assistants have memory limited to the session
    • Between each session or user, everything is "forgotten"
    • The same problem may be solved 10 different ways by 10 team members

The MCP solution explained

  • Architecture: An MCP server exposing a knowledge graph
    • Simple format: storage of triplets (Subject, Predicate, Object)
    • Example: "Our system" "uses" "PostgreSQL"
  • Operation:
    • AIs can query this graph to understand team context
    • They can also add relevant information to it
    • The whole team benefits from individual interactions

The importance of MCP prompts here

Concrete example that illustrates the value of prompts:

  • Without guiding prompt, each AI creates different relations:
    • "PostgreSQL" "is the database of" "Our system"
    • "Our system" "depends on" "PostgreSQL"
    • "We use" "PostgreSQL" "for our project"
  • With appropriate MCP prompts:
    • Controlled vocabulary: only certain predicates are allowed
    • Standardized relations: only "uses", "depends on", etc.
    • Result: coherent graph that can be efficiently queried
